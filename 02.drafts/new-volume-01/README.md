# new-volume-01 本文ドラフト管理メモ

## このフォルダの目的

このフォルダは、新作第1巻の候補本文ドラフトを保管する場所です。

現時点の `chapter-00X.md` は、最終的な章番号を示すものではありません。
元になった事例ファイルの `case_id` に対応させた仮ドラフト名です。

## 現在のドラフト一覧

| ファイル名 | 対応する case_id | 現在の仮章番号 | テーマ |
| --- | --- | --- | --- |
| `chapter-001.md` | `case-001` | 第1章 | 一部売却のつもりが家ごと売却対象になり、媒介報酬トラブルになる話 |
| `chapter-003.md` | `case-003` | 第3章 | タトゥーのある借主への契約後の引渡拒否が、貸主の損害賠償責任につながる話 |
| `chapter-005.md` | `case-005` | 第5章 | ベランダ排水口の清掃不足により、台風時の漏水被害でも借主責任が問題になる話 |

## ファイル名と case_id の関係

- `chapter-001.md` は、`case-001` に基づく本文ドラフトです。
- `chapter-003.md` は、`case-003` に基づく本文ドラフトです。
- `chapter-005.md` は、`case-005` に基づく本文ドラフトです。

ここでの `001`、`003`、`005` は、元事例の番号を引き継いだ管理番号です。
最終章順を示す番号ではありません。

## 現時点の章番号は仮番号

本文中の次の表記は、現時点ではすべて仮番号です。

- `chapter-001.md` 内の「第1章」
- `chapter-003.md` 内の「第3章」
- `chapter-005.md` 内の「第5章」

新作第1巻の最終的な10章構成は、まだ確定していません。
そのため、現在の章番号を前提に目次、章順、ファイル名を確定しないでください。

## 最終章順決定後の整理ルール

最終的な10章構成案を作成した後、必要に応じて次の順番で整理します。

1. 10章分の採用ケースと掲載順を確定する。
2. 各ドラフトの `case_id` と採用章番号の対応表を作成する。
3. 必要に応じて、ファイル名を最終章番号に合わせて変更する。
4. 本文冒頭の見出しと front matter の `chapter` を、最終章番号に合わせて修正する。
5. 目次、章タイトル一覧、関連する管理メモを同じ章番号に統一する。
6. リネーム前後で `case_id` が追跡できるように、対応表を残す。

なお、現時点では本文ドラフトの修正、章番号の変更、ファイル名のリネームは行いません。

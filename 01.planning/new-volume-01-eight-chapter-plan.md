# 新作第1巻 8章構成案

## 結論

- 第1巻は、次の8件で仮固定する。
  1. `case-005`：バルコニー排水口の清掃義務と浸水損害
  2. `case-009`：上階騒音の差止・損害賠償請求棄却
  3. `case-011`：原状回復・敷金精算
  4. `case-006`：民泊禁止特約の告知義務
  5. `case-007`：定期借家契約と媒介業者責任
  6. `case-003`：タトゥーを理由とする契約後の鍵引渡拒否
  7. `case-004`：賃借人行方不明と特約解除・正当事由
  8. `case-008`：無断転貸・無断改装による契約解除
- 現在の `chapter-00X.md` は、最終章番号ではない。元になった `case_id` に対応する仮ドラフト名として扱う。
- 今回は本文ドラフト、README、ファイル名、front matter、本文中の章番号を変更しない。
- `case-001`、`case-002`、`case-010` は第1巻へ無理に入れない。売買寄り・税務寄りの素材を外すことで、巻の軸を守る。
- 章順は「身近な生活トラブルから入り、退去精算、契約前・契約時、契約後対応、管理困難、事業用応用へ進む」構成とする。

## 各章の役割と編集メモ

### 第1章 `case-005`

- 章の役割：第1巻の入口。日常的なベランダ清掃から、借主の善管注意義務、貸主・管理会社の説明記録、保険対応まで広げる。
- 前後章とのつながり：第2章の騒音と並べて、生活トラブルでも「原因・証拠・記録」が結論を分けることを示す。
- 最終編集で注意すべき点：
  - 「台風なら常に借主責任」と読める表現を避ける。
  - 特約、植木鉢、過去の排水管工事、他原因の有無など、この事案で責任が認められた要素を残す。
  - 控訴後和解や保険事情は、本文末か注記で簡潔に扱う。

### 第2章 `case-009`

- 章の役割：生活トラブルの立証回。騒音苦情、測定、発生源、差止め、損害賠償のハードルを整理する。
- 前後章とのつながり：第1章では責任が認められた生活事故、第2章では立証不足で請求が認められなかった生活トラブルとして対比する。第3章の退去精算へ、入居中トラブルから退去後トラブルへ移る。
- 最終編集で注意すべき点：
  - 「うるさいと言えば勝てる」ではなく、測定条件・継続性・現在の侵害を明確にする。
  - `60dB`、`35dB`、ヘッドセット使用、管理組合・仲介者の文脈は、事案の具体性として残す。
  - 文体が硬い箇所は、他章の「です・ます」調に合わせる。

### 第3章 `case-011`

- 章の役割：住宅賃貸の王道テーマである原状回復・敷金精算を担当する。8章構成の不足を補う中核回。
- 前後章とのつながり：第1章・第2章の入居中トラブルから、退去時精算へ移る。第4章以降の契約前・契約時の確認論点に入る前に、読者ニーズの高い実務テーマを押さえる。
- 最終編集で注意すべき点：
  - 更新料、空室損料、室内清掃費、エアコン清掃費を混ぜず、項目別に判断する。
  - 「敷金から何でも引ける」「清掃費は常に無効」と読める表現を避ける。
  - 既刊の敷金・原状回復説明と重なるため、判例ベースの精算実例として差別化する。

### 第4章 `case-006`

- 章の役割：契約前確認と告知義務の応用回。建物そのものではなく、借地契約・管理規約・底地人承諾など外部制限を確認する重要性を示す。
- 前後章とのつながり：第3章までの生活・退去トラブルから、契約前確認へ視点を戻す。第5章の定期借家手続と合わせて「契約書だけでは足りない」ブロックを作る。
- 最終編集で注意すべき点：
  - 民泊一般の可否論に広げすぎない。
  - 本件の要点は、借主の利用目的を把握していたのに、目的利用を妨げる外部制限を確認・告知しなかった点に置く。
  - 借地契約、底地人承諾、管理規約、許認可確認をチェックリスト化する。

### 第5章 `case-007`

- 章の役割：契約時手続の柱。定期借家契約は、契約書や重要事項説明だけでなく、法定の事前説明手続が必要だと示す。
- 前後章とのつながり：第4章の「目的どおり使えるか」から、第5章の「契約を予定どおり終了させられるか」へ進む。第6章の契約後引渡拒否へ、契約成立後の対応リスクを橋渡しする。
- 最終編集で注意すべき点：
  - 重要事項説明と借地借家法38条の事前説明を混同しない。
  - 「同じ紙ならよい」ではなく、説明時期・書面交付・説明権限・記録を分ける。
  - 媒介業者責任と貸主側の確認責任を混ぜず、役割ごとに整理する。

### 第6章 `case-003`

- 章の役割：契約後対応の失敗回。貸主側の不安があっても、契約成立後の一方的な鍵引渡拒否には法的リスクがあると示す。
- 前後章とのつながり：第5章の契約手続から、第6章で契約成立後の対応に移る。第7章の長期不在対応と並べて、感情や不安だけで実力行使しない流れを作る。
- 最終編集で注意すべき点：
  - タトゥーをセンセーショナルに扱わない。
  - 貸主側の不安、反社確認、入居審査の実務背景を説明した上で、契約後拒否の限界を示す。
  - 「見た目で断れない」という単純な話にせず、契約前審査と契約後履行の違いを明確にする。

### 第7章 `case-004`

- 章の役割：管理困難・終了時対応の重い回。長期不在、特約解除、正当事由、立退料、自力救済禁止を扱う。
- 前後章とのつながり：第6章の契約後対応から、管理中・終了時の手続へ進む。第8章の事業用応用回へつなぎ、最後に管理会社・貸主の承諾管理と記録管理を締める。
- 最終編集で注意すべき点：
  - 「長期不在特約があれば即解除できる」と読める表現を避ける。
  - 本件では、特約解除だけでなく、正当事由と立退料を含む事案判断が重要であることを残す。
  - 安否確認、連絡履歴、不在者財産管理人、建替え事情の資料化を具体的に整理する。

### 第8章 `case-008`

- 章の役割：第1巻の応用回。事業用店舗の無断転貸・無断改装を通じて、承諾管理、占有者確認、工事承諾、原状回復の実務を締める。
- 前後章とのつながり：第7章の管理困難・明渡し対応から、第8章で事業用賃貸の承諾管理へ広げる。巻末章として「住宅だけでなく、事業用ではさらに書面管理が重要」とまとめる。
- 最終編集で注意すべき点：
  - 事業用店舗色が強いため、住宅賃貸の基本回ではなく応用回として見せる。
  - 原状回復費用だけに寄せず、誰が使っているか、何を承諾したか、どの工事を許したかに焦点を置く。
  - 文体が硬い箇所と本文中の仮番号メモは、最終編集時に整理する。

## 第1巻 仮タイトル案

1. 女子高生でもわかる賃貸契約と管理トラブル
2. 先生、賃貸トラブルは契約書だけでは終わりません
3. 先生、貸した部屋でこんな揉め方しますか？
4. 賃貸管理の失敗例から学ぶ契約・説明・記録の基本
5. 貸す前・貸した後に揉めないための賃貸トラブル入門

最有力は 1。

- シリーズ名との接続が自然。
- 賃貸契約と賃貸管理トラブルが主題だと一目で分かる。
- 実務者にも一般読者にも広げやすい。

## 想定読者

### 主読者

- 賃貸物件を持つ不動産オーナー
- 自主管理をしている貸主
- 管理会社の新人・若手担当者
- 賃貸仲介会社の営業担当者
- 賃貸管理業務を始めたばかりの実務者

### 副読者

- 借主・入居者
- 事業用物件を借りる個人事業主・法人担当者
- 相続で賃貸物件を引き継いだ人
- 不動産投資を始めたばかりの人
- 賃貸トラブルを一般知識として知りたい読者

### 読者に提供する価値

- 契約前に確認すべき用途制限・外部制限が分かる。
- 契約時に必要な法定手続と記録が分かる。
- 入居中の水漏れ・騒音・長期不在で何を記録すべきか分かる。
- 退去時の敷金精算で、費目ごとに判断する必要が分かる。
- 感情や思い込みではなく、契約・手続・証拠で動く重要性が分かる。

## 8章構成の強み

- 住宅賃貸の入口として強い `case-005`、`case-009`、`case-011` を前半に置ける。
- 生活トラブル、退去精算、契約前確認、契約時手続、契約後対応、管理困難、事業用応用まで一通り並ぶ。
- `case-011` を入れることで、7章案で不足していた「原状回復・敷金精算」を補える。
- 売買寄り・税務寄りの素材を外すため、第1巻のテーマがぶれにくい。
- 8章なら、10章構成より編集負担を抑えつつ、短すぎる印象も避けられる。

## 8章構成の弱み

- 修繕義務・設備故障・賃料減額の章がない。
- 賃料滞納・保証会社・解除明渡しの基本回がない。
- `case-006` は民泊・借地契約、`case-008` は事業用店舗に寄るため、住宅賃貸中心の読者には応用色が強い。
- 第4章以降は制度・手続の話が増えるため、会話劇の軽さを保つ編集が必要。
- 章ごとの教訓が「記録を残す」に寄りやすい。最終編集では、章別の一言を差別化する必要がある。

## 後工程の作業手順

### 1. 章順の最終確認

1. この8章順で、読者入口、退去精算、契約前、契約時、契約後、管理困難、事業用応用の流れに問題がないか確認する。

3. `case-008` を第1巻末に置くか、別巻候補に回すか最終判断する。
4. 修繕義務・滞納対応を入れない8章構成で進めると明記する。

### 2. 本文中の章番号修正

1. 各ドラフト冒頭の `# 第X章：...` を最終章番号に合わせる。
2. 本文中に残っている「第8章表記は仮番号」などのメタ記述を削除または管理メモへ移す。
3. 章内で他章を参照している箇所があれば、最終章番号に合わせて修正する。
4. 章番号だけを直し、本文内容の大幅改稿とは分けて作業する。

### 3. front matter 修正

1. `title` を改善後の章タイトルに更新するか、現タイトルを維持するか決める。
2. `case_id` は変更しない。
3. `chapter` や `chapter_number` がある場合は、最終章番号に合わせる。
4. `style_reference` など欠けている項目があれば、全章で揃える。

### 4. README 修正

1. `02.drafts/new-volume-01/README.md` の「現在のドラフト一覧」を最終章順に更新する。
2. `chapter-00X.md` が case 対応の仮名だった説明を、リネーム後は「旧ファイル名との対応表」に変更する。
3. 第1巻から外す `case-001` の扱いを明記する。
4. READMEには、最終章番号、case_id、旧ファイル名、現ファイル名、テーマを残す。

### 5. 目次作成

1. 章タイトルを本文タイトルと完全一致させる。
2. サブタイトルを入れる場合は、全章で粒度を揃える。
3. 目次では、生活トラブル3章、契約前・契約時2章、契約後・管理終了2章、事業用応用1章の流れが見えるようにする。
